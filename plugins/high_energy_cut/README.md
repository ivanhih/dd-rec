# 高能剪切插件

对已录制的直播录播做二次分析和智能剪辑。

## 功能

- **录播选择**:按主播/直播间筛选已录播,日期选择器自动加载对应 MP4 + 弹幕文件
- **高能片段分析**:基于弹幕密度自动识别"高能片段"
  - 滑动窗口扫弹幕,统计重复 key 数量
  - 合并相邻命中 → 过滤过短 → 输出 Clip 列表
  - 屏蔽词机制(默认屏蔽抽奖/礼物/投喂等)
- **预览**:内置 QMediaPlayer,选中片段可"仅播此片段"
- **批量剪切**:勾选若干片段后一键剪切到 `<录播所在目录>/高能剪切/`
  - 优先 `-c copy`(秒级,无重编码损失)
  - 失败自动回退到 libx264 重编码

## 安装

1. 把整个 `high_energy_cut/` 目录放到 `DDRec/plugins/` 下
2. 或者从"插件商店"一键安装
3. 重启 DDRec
4. 左侧 sidebar 会自动多出一个 🎬 图标,点进去即可

> 目录名必须是 `high_energy_cut`(下划线,跟 Python 模块名规则一致),
> 不能用连字符 —— 插件加载器会用 `importlib.import_module(<id>)` 直接
> 当 Python 模块名 import,带连字符会失败。

## 卸载

进入"插件"页面 → 找到"高能剪切"→ 点"卸载"。

## 设置

进入插件后,在"插件管理"或对应入口打开**专属设置页**:

- 滑动窗口大小(秒)
- 重复弹幕阈值
- 最短片段时长
- 最低密度阈值
- 相邻片段合并间隔
- 屏蔽词列表(一行一个)
- 输出文件命名模板
- 转码质量(CRF)
- 「-c copy 失败时自动回退」开关

设置独立于主程序全局设置,不会污染录播/录制相关配置。

## 算法说明

v1 只做"重复弹幕密度"判断,稳定直观:

1. 解析 XML 弹幕 → `(time, text, uid)` 列表
2. 屏蔽词过滤 + 极短弹幕(< 2 字符)过滤
3. 滑动窗口(默认 10s,步长 1s)扫所有弹幕
4. 归一化(`去空白 + 前 8 字符`)+ 统计窗口内 key 频次
5. 最高频 key 数 ≥ 阈值 → 该窗口记为命中
6. 合并相邻/重叠命中(间隔 < `merge_gap`)→ 一个 Clip
7. 过滤 `duration < min_clip_duration` 的短 Clip

后续计划加入:`KeywordRule`(关键词密度)、`SpeedRule`(语速异常)等扩展规则。

## 输出位置

```
<save_dir>/<uname>/<YYYY-MM-DD>/高能剪切/<原 stem>__片段<N>_<HH-MM-SS>_<HH-MM-SS>.mp4
```

例: `录播文件/张三/2025-12-30/高能剪切/123456-张三-2025-12-30-203015-123__片段1_00-00-23_00-00-45.mp4`

## 性能

- 2 小时录播 1-2w 条弹幕:分析 1-3s(后台线程,不卡 UI)
- 剪切速度取决于片段数量 + ffmpeg 模式(`-c copy` 几秒一个,`libx264` 几秒到几十秒)

## 已知限制

- v1 不支持多 part 拼接分析(只分析该日期下第一个有弹幕的 part)
- v1 不集成到"全局设置"页,设置完全独立
- v1 预览不支持逐帧;`QMediaPlayer` 足够日常使用
